jQuery AJAX初解

  • ajax
前端

AJAX 是一种与服务器交换数据的技术，可以在不用刷新整个页面的情况下更新网页的一部分。在现如今的前端开发中，ajax技术被广泛使用，所以作为一个前端开发人员，ajax是必备的技能。

现在关于ajax的使用语法到处都是，不了解的可以自己百度，本篇文章不作详细说明。只是对一些以前没有接触过前后台交互的新手朋友们，在刚开始使用ajax时可能遇到的问题作以说明，而这也是我自己与一些朋友刚开始所遇到的问题。

jQuery中的AJAX方法有好多，在这里，我就直接拿万能的 $.ajax 说事了，废话不多说，直接上代码图：

麻雀虽小，但五脏俱全，这段代码虽然相当简单，不过不要紧，足够说明问题了，下面就对 $.ajax{} 内的语句逐一分析：

1. url

url 规定请求所要发送的地址，请求分获取请求和提交请求。
通俗点讲，在进行前后台交互的时候，我们需要从后台获取数据，也会提交数据到后台，而不管我们是获取数据还是提交数据，都需要一个地址。获取数据你需要告诉别人你要从哪获取，提交数据你需要告诉别人你要提交到哪，而具体的地址是什么，当后台开发人员写好了，你就知道了。

2. type

type 是你所请求的方式，分为get和post两种方式。
关于get和post两种方法的区别，我在这也不多讲，大家自行百度就好了，另外在你百度的时候可以参考这篇关于get与post的文章 不再以讹传讹，GET和POST的真正区别 。

3. dataType

dataType 规定预期的服务器响应的数据类型，有xml，html，text，script，json，jsonp六种。
这东西没什么好讲的，需要什么类型写什么类型就好。

4. data

data 规定要发送到服务器的数据，也就是服务器需要的东西。
肯定有新手朋友会问，到底服务器需要什么东西，我们要发送什么数据到服务器？举个例子：
如果你需要从服务器中获取东西，就好比你要买一个东西，卖家的仓库就是数据库，你需要卖家从仓库里拿出你想要的东西，你就得告诉他物品的名称或者编号，又或者其他与物品有关的。

在实际开发中，你需要传给后台的数据都会很明显，你从后台程序里就能知道。

5. success

success 当请求成功时运行的函数，另外它有一个参数，function(data)。
