对于前端开发的第一次正式思考与总结

  • css
  • reset
  • 效率
前端

正式进入公司实习已经一个月了，跟我一起进去的还有另外一位，在没进公司之前，以为公司会有一些培训，能学一些东西。然而实际情况是，带我们的那位给了我们一个网站，是网易NEC的，让我们看了一上午NEC的前端规范，然后下午就给了我们俩几十个页面，为了体现一下个人能力，所以选了20多个稍微复杂点的页面，另外20几个登录页面则给了另外一人。从静态页面到jQuery效果，再到ajax交互，这一写就是一个月，在这个过程中学到了一些东西，也遇到一些问题。作为一名刚大三结束的学生，也是第一次参与项目，所以在这把这个月的遇到的问题以及我的思考与总结写出来，留给以后的自己，也希望能对即将正式开始前端工作的其他新手朋友一些帮助，当然如果对于我说出的一些问题，有哪位前辈能给出更好的解决方案那是最好不过的了。^_^

一个项目的开始，动手前的准备

真正做一个项目跟我们平时自己练习时写一些单独的页面是有很大差别的，不是技术上有多大差别，而是对于具体流程和项目整体的思考。

在平时练习写一些单独的页面的时候，很多人应该是拿到PSD图或者JPEG图就直接开始写了，好一点的会想想页面布局，或者动手画一画这个页面分哪几个块。而开发一个整体项目的时候，我们要做的有更多：

1. 把你要写的页面搞明白

如果你不是参与项目设计的人员之一，那么你一定要搞清楚你所拿到的几十个页面之间的关系，以及搞明白页面的整体结构和具体效果。我们刚开始写的时候，负责人没给我们说明白，我们没有经验也没怎么问，到了后来静态页面都写完了，开始写jQuery效果的时候才发现当初做了好些无用功，还有一些页面结构存在问题，但是我不明白的是负责人让我们每天把写的东西都交给他们看看，可他们却什么也没说，就说了一句写的还不错，继续加油……所以对于其他新手朋友的建议是，当你拿到页面试样的时候，一定要搞明白页面的结构、效果和主要功能。

2. 记住，你不再是一个人

项目开发和平时写deme是绝对不一样的，如果你的能力很强，你大可以提一些非常棒的建议，或者帮助团队里的其他成员解决一些技术上的问题，但是一定要记住，你现在得和别人协同工作，切忌特立独行。

当初我们在开始项目的时候，在这方面做的还算不错，但是现在看起来依旧不令人满意，反正我自己是不太满意。

文章的开始就说了，带我们两位新手的负责人就给了我们半天时间看网易NEC的前端规范，然后下午讲了讲都有些什么功能的页面，之后就让我们开始写了代码了。

我跟另外一位新人一起讨论了下如何创建项目，便于以后管理和文件放在一起后不会发生路径上的错误，然而一直到静态页面写完才知道，我们后台语言用的是Groovy，然后生产过程用的是Grails(一套基于Groovy编程语言,用于快速Web应用开发的开源框架)，既可以写后台，又可以用作项目管理，所以我们商量的如何创建项目，其实没什么用，但是对于不使用这个框架的公司，我觉得应该还是有必要的。我个人是学硬件的，所以根本就没听过这个什么Groovy，后来一问才知道，整个大连用这个的不超过5家，真不知道他们怎么会选使用率这么低的框架。

除了关于怎么创建项目，我们俩还讨论了怎么给class命名，防止以后将所有CSS放在一起后会产生冲突。我们当时的做法是加前缀，每个页面的class命名分别加上对应的前缀，这样的做法的确可行，当我们把所有CSS直接放在一个页面后，并没有出现什么问题。

但是现在想来这个做法还是欠妥，因为有很多页面的模块样式是可以重用的，但由于我们在写代码之前也没有仔细分析过每个页面结构，以我们给这些可以重用的样式依旧加上了一个页面所对应的前缀，也就是说我们在页面B的样式文件里通常会加上一些带有页面A的前缀的样式，代码一多，把我们自己都搞糊涂，甚至后来把所有CSS放在一起时，写注释都不知道怎么写。

所有建议大家在动手前，一定要跟你的队友讨论好你们在开发过程中可能遇到的问题，并且制定好一个规范，要懂得怎么合作。

是时候动手了

跟你的队友搞明白项目的整体结构，弄清楚每个页面的功能和效果，商量好如何创建并管理项目，那么接下来，是时候开始动手了。

1. 创建项目

就这次项目经历来看，如果你们在开发过程中并没有使用到sass和compass之类的东西，就直接写HTML+CSS+JavaScript，那么你们只需要3个文件夹，分别是css文件夹，images文件夹，JavaScript文件夹，HTML文件直接放在根目录就行，不需要再单独去建立对应功能页面的文件夹，如果用到icon-font的话，再建个fonts文件夹用来放字体文件就行。别的我不知道，但至少如果你们也用Grails框架或者用的是dede这样的CMS建站系统，就这样就行了。

CSS文件的话，我建议是3个，分别是reset.css,common.css,page.css。这也是很多前人所采用的，一个用来放reset样式，一个用来放可以共用的模块样式，一个用来放独立的页面样式。

JavaScript文件的话，一个Plugin.js，然后每个页面一个base.js，因为我觉得JavaScript最好就别放在一起了，容易冲突，还不好修改，除非你的页面所用到的JavaScript很少，而且几乎没有冲突的可能，那你可以放在一起。

用一个图来表示：

|-- css
    |-- reset.css
    |-- common.css
    |__ page.css
|-- images 
|-- js
    |-- plugin.js
    |-- page1.js
    |__ page2.js
|-- page1.html
|__ page2.html

这里只是给大家展示了一个常用的，不同的公司会用到不同的工具，一些框架，一些自动化管理工具，或者sass之类的预处理器，所以最终结果也会有些差别，大家灵活运用就好。
